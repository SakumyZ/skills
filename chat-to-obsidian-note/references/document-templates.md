## 通用 Frontmatter

所有文档必须包含以下 frontmatter，与知识库现有模板保持一致：

```yaml
---
tags: [{分类名}]
parent: {父级分类名}
author: SakumyZ
createTime: {YYYY-MM-DD HH:mm:ss}
modifiedTime: {YYYY-MM-DD HH:mm:ss}
---
```

**字段说明：**

| 字段 | 说明 | 示例 |
|------|------|------|
| `tags` | 文件所在目录名，数组格式 | `[Vue]`、`[Engineering]`、`[Git]` |
| `parent` | 父级目录名 | `Front-end`、`Web`、`Readme` |
| `author` | 固定为 `SakumyZ` | `SakumyZ` |
| `createTime` | 文档创建时间 | `2026-02-22 14:30:00` |
| `modifiedTime` | 文档修改时间，首次与 createTime 相同 | `2026-02-22 14:30:00` |

**嵌套目录的 tags/parent 示例：**

| 文件路径 | tags | parent |
|---------|------|--------|
| `Git/xxx.md` | `[Git]` | `Git` |
| `Web/Front-end/Vue/xxx.md` | `[Vue]` | `Front-end` |
| `Web/Front-end/Engineering/xxx.md` | `[Engineering]` | `Front-end` |
| `Web/Backend/Nest/xxx.md` | `[Nest]` | `Backend` |
| `Ohters/Obsidian/xxx.md` | `[Obsidian]` | `Ohters` |

## 章节编写指南

### 背景

- 提供足够的上下文，让读者不看聊天记录也能理解
- 问题解决型：描述在什么场景下做什么操作时遇到了什么问题
- 知识学习型：说明学习动机和应用场景
- 方案对比型：说明业务需求和约束条件

### 环境信息

- 只列出与问题直接相关的环境要素
- 如果对话中未提及环境信息，尝试从代码片段中推断（如 import 语句可推断框架版本）
- 信息不足时可省略此节，或标注「未确认」

### 关键代码 / 代码示例

- 代码块必须标注语言（```typescript、```vue、```bash 等）
- 问题解决型：展示修改前后对比，用 `❌` 和 `✅` 标注
- 知识学习型：提供从简到繁的递进示例
- 方案对比型：每个方案提供最小可运行示例
- 去除对话中的调试/试错代码，只保留最终版本

### 注意事项

- 提炼对话中提到的「坑」和 edge case
- 补充可能的副作用或影响范围
- 列出未来可能需要注意的点

### 参考资料

- 优先使用对话中提到的链接
- 链接格式：`[标题](URL)`，不使用 Obsidian `[[]]` 格式
- 只包含真实有效的链接，不编造 URL
- 如果对话中没有提供链接，可基于话题推荐官方文档链接

### 相关知识

- 使用 Obsidian `[[文件名]]` 双链格式
- **只引用知识库中实际存在的文件**
- 通过扫描目标分类目录获取已有文件列表
- 如果找不到相关文件，可以留空此节或写「暂无关联笔记」
