## 分类决策规则

1. **关键词匹配优先级**：子分类 > 父分类。例如内容关于 Vue3 应优先匹配 `Web/Front-end/Vue/` 而非 `Web/Front-end/`。

2. **Trouble Shooting 子目录**：如果文档类型是「问题解决型」，且属于 Vue 或 React 领域，优先考虑 `Trouble Shooting/` 子目录。通用前端问题使用 `Front-end Trouble Shooting/`。

3. **多主题交叉**：当内容横跨多个分类时，按**核心解决的问题**所属领域归类。例如「在 Vue 项目中配置 Vite」→ `Web/Front-end/Engineering/`（核心是 Vite 配置）。

4. **无法确定时**：默认分类到 `Ohters/`，并在向用户确认时说明原因。

5. **新分类**：如果现有分类都不匹配，允许用户指定新路径，将在知识库中创建新目录。

### 中间层索引文件

部分目录充当纯导航层，其索引文件只包含指向子分类的 wiki link，无 frontmatter 和标题分组：

| 中间层索引                               | 内容示例                                                     |
| ---------------------------------------- | ------------------------------------------------------------ |
| `Web/Front-end/Front-end.md`             | `[[Foundation]]` `[[Vue]]` `[[React]]` `[[Engineering]]` ... |
| `Web/Front-end/Foundation/Foundation.md` | `[[CSS]]` `[[HTML]]` `[[JavaScript]]` `[[TypeScript]]`       |

当新建子分类时（如 `Web/Front-end/Svelte/`），需要在对应的中间层索引中追加 `[[Svelte]]`。

### 格式规范

根据对现有索引文件的分析，格式规范如下：

1. **Wiki link 写法**：每行一个裸 `[[文件名]]`，**不加 `-` bullet**
2. **分组方式**：使用 `## 二级标题` 对笔记进行主题分组
3. **分隔线**：分组之间可选用 `---` 分隔（与该索引文件现有风格保持一致）
4. **Frontmatter**：索引文件的 tags 通常包含 `Index` 标签，如 `tags: [Index, Vue]`
5. **追加位置**：新 wiki link 添加到最匹配的 `##` 分组末尾；无匹配分组时追加到文件末尾
